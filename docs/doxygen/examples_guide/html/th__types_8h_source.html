61/*------------------------------------------------------------------------------
62 * The following data types are called the "Standard EEMBC Data Types". Each
63 * is defined appropriatly for the target exeuction environment and the
64 * tool chain used to compile/link and possibly locate an instance of the
65 * Test Harness and one or more benchmarks.
66 *
67 * Each of these data types MUST BE defined in a manner which gives
68 * them a size of <<AT LEAST>> the number of bits indicated by its name.
69 *
70 * It is every important to note that a type MUST NOT be defined larger than
71 * specified if the build and execution environments support the appropriatly
72 * sized data type.
73 *
74 * For example, if a 'short' is 16bits on a target and an 'int' is 32bits,
75 * then the 'e_u16' data type must be defined as 'unsigned short' and
76 * not 'unsigned int'.
77 *
78 * The 8bit, 16bit, 24bit and 32bit integral data types are required.
79 * The 48bit and 64bit integral types are optional.
80 * The 32bit and 64bit floating point types are optional
81 * ---------------------------------------------------------------------------*/
