51/* Work item connection related functions.
52 These helper function are a way to use the connections to synchronize items.
53 The protocol works like this:
54 Item A --> Item B
55 Two connections are needed. The first connection is used to control B, the second is used to throttle A and pass data.
56 The first connection is referred to as CTRL.
57 The second connection is referred to as DATA.
58 The data on CTRL connection is initialized by A, and becomes "QUIT" when A is done working.
59 A owns the mutex on CTRL, and B owns the mutex on DATA.
60
61 Before starting work, A needs to init_ctrl() and B needs to lock_in().
62 Then, B needs to:
63 wait_prev_start(map,connections,channel);
64 and:
65 while (signal_in(map,connections,channel) != QUIT) {
66 wait_prev_ready(map,connections,channel);
67 buf=get_next(map,connections,channel);
68 .. do work on buf
69 }
70 A needs to:
71 ... create a buffer
72 wait_next_ready(map,connections,channel);
73 set_out(map,connections,buf,channel);
74 signal_out_ready(map,connections,channel);
75 And when done:
76 signal_out(map,connections,QUIT,channel);
77 wait_next_ready(map,connections,channel);
78*/
